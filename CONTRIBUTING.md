## Code of conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating you agree to abide by its terms.

## Getting started

1. **Fork** the repository and clone your fork.
2. Open the project in the dev container (**Dev Containers: Reopen in Container**) — this gives you the exact same environment used in CI.
3. Make your changes in a feature branch: `git checkout -b feat/my-change`.

## Reporting bugs

Use the [Bug Report](.github/ISSUE_TEMPLATE/bug_report.md) issue template. Please include:

- VS Code and Dev Containers extension versions
- Docker Desktop version
- The full text of any error from **Output → Dev Containers**
- Steps to reproduce

## Requesting features

Use the [Feature Request](.github/ISSUE_TEMPLATE/feature_request.md) issue template. Describe the problem you are trying to solve, not just the solution.

## Submitting a pull request

1. Open an issue first for non-trivial changes so we can align on direction.
2. Keep PRs focused — one logical change per PR.
3. Update `README.md` and `AGENTS.md` if your change affects the environment or toolchain.
4. Update `CHANGELOG.md` under the `[Unreleased]` section.
5. Ensure the dev container still builds cleanly after your changes.
6. Fill in the [pull request template](.github/PULL_REQUEST_TEMPLATE.md).

## Development guidelines

### devcontainer.json

- Prefer **features** over custom Dockerfile layers for runtime dependencies.
- Pin features via `devcontainer-lock.json` — run `devcontainer upgrade` to update.
- Use `docker-outside-of-docker` (not `docker-in-docker`) to avoid startup timeouts. See [README.md](README.md#why-docker-outside-of-docker-instead-of-docker-in-docker) for rationale.

### setup.sh

- The script runs **once** on container creation (`postCreateCommand`). Keep it idempotent.
- Network downloads should be guarded with existence checks (e.g. `if ! command -v copilot`).
- Additional optional setup lives in `.devcontainer/scripts/` — do not add non-essential steps to `setup.sh` itself.

### MCP servers

- `.mcp.json` is the **single source of truth**. Do not edit `.vscode/mcp.json` directly — it is generated by `setup.sh` from `.mcp.json`.

### Commit style

This project uses [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add XYZ feature
fix: correct ABC bug
docs: update README
chore: bump devcontainer feature versions
```
